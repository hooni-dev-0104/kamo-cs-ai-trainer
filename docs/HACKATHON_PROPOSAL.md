# CS AI 트레이너 기획서

## 문제 정의

고객 서비스 상담원은 업무에 투입되기 전 실전 대응 능력과 이론 지식을 모두 갖춰야 한다. 신입 상담원은 화난 고객 응대, 돌발 상황 처리, 회사 정책 숙지 등 복합적인 역량이 필요하지만, 기존 교육 방식은 시니어 상담원의 일대일 멘토링과 집체 교육에 의존한다. 이 과정에서 교육 비용과 시간이 과도하게 소요되며, 신입 상담원이 실전에 투입될 때까지 실제 고객과의 대화 경험을 충분히 쌓지 못하는 문제가 발생한다.

## 페인포인트

시니어 상담원 1명이 신입 1명을 멘토링하는 데 주당 10시간 이상이 소요되며, 이는 시니어의 업무 효율을 크게 저하시킨다. 멘토마다 교육 방식과 피드백 기준이 달라 신입 상담원이 받는 교육의 품질이 일관되지 않다. 롤플레이 교육은 일주일에 1-2회만 진행되며, 피드백은 멘토의 기억과 메모에 의존하여 1-2일 후에 제공된다. 학습 자료를 기반으로 이론 평가 문제를 출제하는 작업은 강사가 자료당 2-3시간을 투입해야 하며, 자료가 업데이트될 때마다 문제를 새로 작성해야 한다. 신입 상담원은 실전 투입 전까지 평균 10회 미만의 롤플레이만 경험하며, 이는 다양한 고객 유형과 돌발 상황에 대처하기에 부족한 경험치다. 멘토링 비용과 교육 기간 지연으로 인해 신입 1명당 교육 비용이 수백만원에 달하며, 소규모 기업은 체계적인 교육 시스템을 운영하기 어렵다.

## 솔루션

CS AI 트레이너는 Google Cloud의 Speech-to-Text, Text-to-Speech, Gemini API를 활용하여 AI 고객과의 실전 음성 대화 시뮬레이션과 학습 자료 기반 자동 문제 출제를 제공한다. 상담원은 브라우저에서 시나리오를 선택하고, Text-to-Speech API가 생성한 고객 음성을 듣는다. 이때 SSML 태그를 사용하여 화난, 답답한, 슬픈 등 감정 상태를 음성에 반영한다. 상담원이 마이크로 응답하면 MediaRecorder API가 음성을 녹음하고, Speech-to-Text API가 이를 텍스트로 변환한다. 변환된 텍스트는 Gemini API에 전달되어 대화 맥락을 분석하고, AI 고객의 다음 응답을 생성한다. 대화가 종료되면 Gemini API가 응대 내용을 공감 능력, 문제 해결 능력, 전문성, 어조 4개 항목으로 평가하고 개선 제안을 제공한다. 학습 자료 평가 기능은 관리자가 Zip 파일을 업로드하면 JSZip 라이브러리로 압축을 해제하고, PPTX 파일은 DOMParser로 XML을 파싱하여 슬라이드별 텍스트를 추출한다. 추출된 텍스트는 Gemini API에 전달되며, 관리자가 설정한 문항 수와 필수 포함 키워드, 커스텀 프롬프트를 반영하여 객관식과 O/X 문제를 자동 생성한다. 사용자 인증은 Supabase Auth로 처리하며, 모든 세션 데이터와 피드백은 Supabase PostgreSQL에 저장된다. Row Level Security 정책으로 사용자는 본인 데이터만, 관리자는 전체 데이터를 조회할 수 있다. React와 TypeScript로 구현된 프론트엔드는 Vite로 빌드하여 Vercel에 배포한다.

## 솔루션의 기대 효과

상담원은 24시간 언제든지 무제한으로 AI 고객과 대화 연습을 할 수 있으며, 대화 종료 후 즉시 다차원 평가와 개선 제안을 받는다. 시니어 상담원의 멘토링 시간이 감소하여 본업에 집중할 수 있고, 모든 신입 상담원이 동일한 기준으로 평가되고 피드백을 받는다. 학습 자료를 업로드하면 Gemini API가 수 분 내에 문제를 출제하므로, 강사가 문제 출제에 투입하는 시간을 제거한다. 자료가 업데이트되어도 다시 업로드하기만 하면 새로운 문제가 자동 생성되며, 관리자가 커스텀 프롬프트로 출제 방향을 지정할 수 있다. 신입 상담원은 실전 투입 전 수십 회 이상 다양한 시나리오를 연습하여 경험치를 축적하고, 배지와 리더보드를 통해 학습 동기를 유지한다. Google Cloud API의 무료 티어를 활용하여 초기 도입 비용을 최소화하고, Supabase의 무료 플랜으로 데이터베이스와 인증을 운영한다.

## AI의 역할

Google Gemini API는 세 가지 역할을 수행한다. 첫째, 대화형 AI 고객으로 작동하여 상담원의 응답에 따라 다음 대사를 생성한다. generateContent 메서드로 대화 히스토리와 시나리오 정보를 전달하며, temperature는 0.7로 설정하여 자연스러운 변화를 유도한다. 둘째, 학습 자료를 분석하여 시험 문제를 자동 출제한다. 자료 텍스트와 함께 총 문항 수, 객관식/O/X 비율, 필수 키워드, 커스텀 프롬프트를 JSON 형식으로 전달하고, 응답은 순수 JSON 형식의 문제 세트를 반환한다. 셋째, 상담원의 응대 내용을 평가하고 피드백을 생성한다. 대화 전체 내역을 전달하면 공감 능력, 문제 해결 능력, 전문성, 어조를 각각 점수화하고, 잘한 점과 개선할 점을 구체적으로 제시한다. 퀴즈 결과 분석 시에는 틀린 문제별로 오답 이유, 핵심 개념, 학습 팁을 생성하고, 전체 취약점을 우선순위별로 정리하여 개선 계획을 제안한다. Google Cloud Speech-to-Text API는 상담원의 음성을 텍스트로 변환한다. RecognitionConfig에서 languageCode를 ko-KR로 설정하고, encoding은 LINEAR16, sampleRateHertz는 16000을 사용한다. 변환된 텍스트는 Gemini API에 전달되어 대화 맥락을 이해하는 데 활용된다. Google Cloud Text-to-Speech API는 AI 고객의 대사를 음성으로 변환한다. SynthesizeSpeechRequest에 텍스트와 SSML을 전달하며, SSML의 prosody 태그로 rate, pitch, volume을 조정하여 감정을 표현한다. languageCode는 ko-KR, name은 ko-KR-Standard-A 음성을 사용하고, audioEncoding은 MP3로 설정한다. 생성된 오디오는 base64로 디코딩하여 Blob으로 변환하고, HTMLAudioElement로 재생한다.
